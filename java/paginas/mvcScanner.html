MVC Completo com Scanner

Agora vamos integrar todos os conceitos aprendidos em uma única estrutura: Model, View, Controller e DAO.

O que é MVC?

MVC é um padrão de arquitetura que separa o sistema em três partes:

  • Model → dados
  • View → interface
  • Controller → controle

Fluxo da aplicação


Main → View → Controller → DAO → Dados

Cada camada possui uma responsabilidade específica.

View (Interface)

Responsável por mostrar o menu e coletar dados do usuário. 


import java.util.Scanner;

public class PessoaView {

    private Scanner sc = new Scanner(System.in);

    public int mostrarMenu(){
        System.out.println("1 - Adicionar");
        System.out.println("2 - Listar");
        System.out.println("0 - Sair");
        return sc.nextInt();
    }

    public String lerNome(){
        System.out.print("Nome: ");
        return sc.next();
    }

    public int lerIdade(){
        System.out.print("Idade: ");
        return sc.nextInt();
    }
}

Controller

Recebe os dados da View e chama o DAO. 


public class PessoaController {

    private PessoaDAO dao = new PessoaDAO();

    public void adicionar(String nome, int idade){
        dao.adicionar(new Pessoa(nome, idade));
    }

    public void listar(){
        for(Pessoa p : dao.listar()){
            System.out.println(p.getNome() + " - " + p.getIdade());
        }
    }
}

Main (Integração)

O Main conecta todas as partes do sistema. 


public class Main {

    public static void main(String[] args){

        PessoaController controller = new PessoaController();
        PessoaView view = new PessoaView();

        int opcao;

        do {
            opcao = view.mostrarMenu();

            switch(opcao){

                case 1:
                    String nome = view.lerNome();
                    int idade = view.lerIdade();
                    controller.adicionar(nome, idade);
                    break;

                case 2:
                    controller.listar();
                    break;
            }

        } while(opcao != 0);
    }
}

Entendendo o fluxo

  • Usuário interage com a View
  • View envia dados para o Controller
  • Controller chama o DAO
  • DAO manipula os dados

Resumo

  • MVC organiza o sistema
  • Separa responsabilidades
  • Facilita manutenção e evolução
