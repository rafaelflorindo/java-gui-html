Controller com DAO (MySQL)

Agora vamos evoluir nossa aplicação utilizando um banco de dados real (MySQL).

A principal ideia é perceber que o Controller continua praticamente o mesmo. Apenas o DAO muda, pois agora os dados são armazenados no banco.

Fluxo da aplicação


Main → Controller → DAO → Banco de Dados

O fluxo permanece o mesmo, mudando apenas a fonte de dados.

Classe de Conexão

Responsável por realizar a conexão com o banco de dados. 


import java.sql.Connection;
import java.sql.DriverManager;

public class Conexao {

    public static Connection conectar() throws Exception {
        String url = "jdbc:mysql://localhost:3306/teste";
        String user = "root";
        String password = "";

        return DriverManager.getConnection(url, user, password);
    }
}

Essa classe centraliza a conexão, facilitando reutilização.

Classe DAO (com banco)

Agora o DAO utiliza comandos SQL para acessar os dados. 


import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

public class PessoaDAO {

    // CREATE
    public void inserir(Pessoa p) throws Exception {

        Connection conn = Conexao.conectar();

        String sql = "INSERT INTO pessoa (nome, idade) VALUES (?, ?)";

        PreparedStatement stmt = conn.prepareStatement(sql);
        stmt.setString(1, p.getNome());
        stmt.setInt(2, p.getIdade());

        stmt.execute();
        conn.close();
    }

    // READ (listar)
    public void listar() throws Exception {

        Connection conn = Conexao.conectar();

        String sql = "SELECT * FROM pessoa";

        PreparedStatement stmt = conn.prepareStatement(sql);
        ResultSet rs = stmt.executeQuery();

        while(rs.next()){
            System.out.println(rs.getInt("id") + " - " +
                               rs.getString("nome") + " - " +
                               rs.getInt("idade"));
        }

        conn.close();
    }

    // READ (buscar um)
    public Pessoa buscarUm(int id) throws Exception {

        Connection conn = Conexao.conectar();

        String sql = "SELECT * FROM pessoa WHERE id = ?";

        PreparedStatement stmt = conn.prepareStatement(sql);
        stmt.setInt(1, id);

        ResultSet rs = stmt.executeQuery();

        Pessoa pessoa = null;

        if(rs.next()){
            pessoa = new Pessoa(
                rs.getString("nome"),
                rs.getInt("idade")
            );
        }

        conn.close();

        return pessoa;
    }

    // UPDATE
    public void atualizar(int id, Pessoa p) throws Exception {

        Connection conn = Conexao.conectar();

        String sql = "UPDATE pessoa SET nome = ?, idade = ? WHERE id = ?";

        PreparedStatement stmt = conn.prepareStatement(sql);
        stmt.setString(1, p.getNome());
        stmt.setInt(2, p.getIdade());
        stmt.setInt(3, id);

        stmt.execute();

        conn.close();
    }

    // DELETE
    public void excluir(int id) throws Exception {

        Connection conn = Conexao.conectar();

        String sql = "DELETE FROM pessoa WHERE id = ?";

        PreparedStatement stmt = conn.prepareStatement(sql);
        stmt.setInt(1, id);

        stmt.execute();

        conn.close();
    }
}

O DAO agora depende de SQL e conexão com banco de dados.

Classe Controller

O Controller continua com a mesma responsabilidade: controlar o fluxo da aplicação. 


public class PessoaController {

    private PessoaDAO dao = new PessoaDAO();

    public void adicionar(String nome, int idade) throws Exception {
        Pessoa p = new Pessoa(nome, idade);
        dao.inserir(p);
    }

    public void listar() throws Exception {
        dao.listar();
    }

    public Pessoa buscar(int id) throws Exception {
        return dao.buscarUm(id);
    }

    public void atualizar(int id, String nome, int idade) throws Exception {
        Pessoa p = new Pessoa(nome, idade);
        dao.atualizar(id, p);
    }

    public void excluir(int id) throws Exception {
        dao.excluir(id);
    }
}

Observe que a lógica é praticamente a mesma da versão com ArrayList.

Isso mostra a vantagem da separação de responsabilidades.

Classe Principal

A classe principal continua interagindo apenas com o Controller. 


public class Main {

    public static void main(String[] args) throws Exception {

        PessoaController controller = new PessoaController();

        controller.adicionar("Carlos", 25);
        controller.adicionar("Ana", 30);

        controller.listar();

        Pessoa p = controller.buscar(1);
        System.out.println("Buscado: " + p.getNome());

        controller.atualizar(1, "Carlos Silva", 26);

        controller.excluir(2);

        System.out.println("Após alterações:");
        controller.listar();
    }
}

O Main não precisa saber se os dados estão em um ArrayList ou no banco.

Resumo

  • Controller não depende da fonte de dados
  • DAO pode ser alterado sem impactar o sistema
  • MySQL permite persistência real
  • Separação de responsabilidades facilita manutenção
