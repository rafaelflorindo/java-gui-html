Organização do Código

Organizar o código é fundamental para facilitar a leitura, manutenção e evolução do sistema.

Programas desorganizados se tornam difíceis de entender e modificar.

Problema comum

No início, é comum colocar toda a lógica dentro do método main. 


public class Exemplo {

    public static void main(String[] args) {

        System.out.println("Bem-vindo!");

        int a = 10;
        int b = 20;

        int soma = a + b;

        System.out.println("Resultado: " + soma);

    }

}

Apesar de funcionar, esse tipo de código não é organizado.

Solução: uso de métodos

Podemos dividir o programa em métodos, onde cada parte tem uma responsabilidade. 


public class Exemplo {

    public static void main(String[] args) {

        mostrarMensagem();
        calcularSoma();

    }

    public static void mostrarMensagem(){
        System.out.println("Bem-vindo!");
    }

    public static void calcularSoma(){
        int a = 10;
        int b = 20;

        int soma = a + b;

        System.out.println("Resultado: " + soma);
    }

}

Separando responsabilidades

Cada método deve ter uma função específica.

  • Entrada de dados
  • Processamento
  • Exibição de resultados

Isso torna o código mais organizado e fácil de entender.

Exemplo mais organizado


public class Exemplo {

    public static void main(String[] args) {

        int resultado = somar(10, 20);
        exibirResultado(resultado);

    }

    public static int somar(int a, int b){
        return a + b;
    }

    public static void exibirResultado(int valor){
        System.out.println("Resultado: " + valor);
    }

}

Vantagens da organização

  • Código mais limpo
  • Facilidade de manutenção
  • Reutilização de métodos
  • Melhor entendimento

Preparando para projetos maiores

Em sistemas maiores, o código é dividido em diferentes classes, cada uma com sua responsabilidade.

Essa organização é a base para padrões como Controller e DAO, que serão estudados posteriormente.

Boas práticas

  • Evitar colocar toda a lógica no main
  • Criar métodos com nomes claros
  • Cada método deve ter uma única responsabilidade

Atividade
