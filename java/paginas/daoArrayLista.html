DAO com ArrayList

O padrão DAO (Data Access Object) é utilizado para separar a lógica de acesso aos dados da lógica da aplicação.

Neste exemplo, utilizamos um ArrayList para simular um banco de dados em memória.

Classe Model


public class Pessoa {

    private String nome;
    private int idade;

    // Construtor
    public Pessoa(String nome, int idade){
        this.nome = nome;
        this.idade = idade;
    }

    // Getters
    public String getNome() {
        return nome;
    }

    public int getIdade() {
        return idade;
    }
}

Classe DAO


import java.util.ArrayList;

public class PessoaDAO {

    private ArrayList lista = new ArrayList<>();

    // CREATE
    public void adicionar(Pessoa p){
        lista.add(p);
    }

    // READ
    public ArrayList listar(){
        return lista;
    }

    // UPDATE
    public void atualizar(int index, Pessoa p){
        lista.set(index, p);
    }

    // DELETE
    public void remover(int index){
        lista.remove(index);
    }
}

Classe Principal (Teste)


public class Main {

    public static void main(String[] args){

        PessoaDAO dao = new PessoaDAO();

        dao.adicionar(new Pessoa("Carlos", 25));
        dao.adicionar(new Pessoa("Ana", 30));

        for(Pessoa p : dao.listar()){
            System.out.println(p.getNome() + " - " + p.getIdade());
        }

    }
}

Explicação

  • ArrayList simula um banco de dados
  • DAO centraliza operações CRUD
  • Separação entre dados e lógica

Resumo

  • DAO organiza o acesso aos dados
  • ArrayList permite testes rápidos
  • Não há persistência real

Atividade

Crie um DAO para a classe Produto contendo:

  • Adicionar produto
  • Listar produtos
  • Remover produto
