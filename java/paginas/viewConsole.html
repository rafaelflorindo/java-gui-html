Linguagem Java

View (Interface no Console)

Até agora, utilizamos o System.out.println() e o Scanner diretamente no código principal.

Porém, em sistemas organizados, a parte responsável por conversar com o usuário deve ficar separada. Essa parte é chamada de View.

O que é a View?

A View é responsável pela interação com o usuário, ou seja:

  • Exibir informações
  • Ler dados digitados

Ela não deve conter regras de negócio e nem acessar o banco de dados.

Antes (forma não organizada)


Scanner sc = new Scanner(System.in);

System.out.print("Nome: ");
String nome = sc.nextLine();

Aqui estamos misturando interface com lógica do sistema.

Depois (usando View)

Agora vamos separar essa responsabilidade em uma classe específica. 


import java.util.Scanner;

public class PessoaView {

    private Scanner sc = new Scanner(System.in);

    public String lerNome(){
        System.out.print("Digite o nome: ");
        return sc.nextLine();
    }

    public int lerIdade(){
        System.out.print("Digite a idade: ");
        return sc.nextInt();
    }

    public void mostrarPessoa(String nome, int idade){
        System.out.println("Nome: " + nome);
        System.out.println("Idade: " + idade);
    }

    public void mostrarMensagem(String msg){
        System.out.println(msg);
    }
}

Explicação do código

  • Scanner → utilizado para entrada de dados
  • lerNome() → solicita o nome ao usuário
  • lerIdade() → solicita a idade
  • mostrarPessoa() → exibe dados na tela
  • mostrarMensagem() → exibe mensagens gerais

Importante

A View não deve:

  • Salvar dados
  • Aplicar regras de negócio
  • Acessar o DAO

Resumo

  • View → interface com o usuário
  • Controller → controla a aplicação
  • DAO → acessa os dados
