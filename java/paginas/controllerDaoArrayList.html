Controller com DAO (ArrayList)

Nesta etapa, vamos organizar melhor nossa aplicação utilizando o padrão Controller + DAO.

O objetivo é separar responsabilidades, deixando o código mais organizado, reutilizável e fácil de manter.

Fluxo da aplicação


Main → Controller → DAO → Dados (ArrayList)

Cada parte possui uma responsabilidade:

  • Main → inicia o sistema
  • Controller → controla as ações
  • DAO → acessa os dados
  • ArrayList → simula um banco

Classe Model

Representa os dados da aplicação. 


public class Pessoa {

    private String nome;
    private int idade;

    public Pessoa(String nome, int idade){
        this.nome = nome;
        this.idade = idade;
    }

    public String getNome() {
        return nome;
    }

    public int getIdade() {
        return idade;
    }
}

A classe Pessoa armazena apenas os dados, sem regras de negócio.

Classe DAO

O DAO é responsável por manipular os dados. 


import java.util.ArrayList;

public class PessoaDAO {

    private ArrayList<Pessoa> lista = new ArrayList<>();

    public void adicionar(Pessoa p){
        lista.add(p);
    }

    public ArrayList<Pessoa> listar(){
        return lista;
    }

    public void remover(int index){
        lista.remove(index);
    }
}

Aqui utilizamos um ArrayList para simular um banco de dados.

O DAO não sabe nada sobre o usuário ou regras, apenas manipula os dados.

Classe Controller

O Controller é responsável por controlar o fluxo da aplicação. 


public class PessoaController {

    private PessoaDAO dao = new PessoaDAO();

    public void adicionar(String nome, int idade){
        Pessoa p = new Pessoa(nome, idade);
        dao.adicionar(p);
    }

    public void listar(){
        for(Pessoa p : dao.listar()){
            System.out.println(p.getNome() + " - " + p.getIdade());
        }
    }

    public void remover(int index){
        dao.remover(index);
    }
}

O Controller cria objetos, aplica regras e utiliza o DAO.

Ele funciona como uma ponte entre o sistema e os dados.

Classe Principal

A classe principal apenas inicia o sistema e chama o Controller. 


public class Main {

    public static void main(String[] args){

        PessoaController controller = new PessoaController();

        controller.adicionar("Carlos", 25);
        controller.adicionar("Ana", 30);

        controller.listar();

        controller.remover(0);

        System.out.println("Após remoção:");
        controller.listar();
    }
}

Observe que o Main não acessa o DAO diretamente.

Isso mantém o sistema organizado e facilita manutenção.

Resumo

  • Model → representa os dados
  • DAO → acessa os dados
  • Controller → controla a aplicação
  • Main → inicia o sistema
